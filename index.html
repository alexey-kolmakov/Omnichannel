📲  Команді для Thunar

Категория Подкатегория Опис Команда
🗒  Файли Права Зробити файл виконуваним 
chmod +x "%f"
🗒  Файли Перейменування Пробіли → підкреслення 
mv "%f" "$(echo "%f" | tr ' ' '_')"
🗒  Файли Симлінки Створити сімлінк 
ln -s "%f" "%n.link"
🗒  Файли Архіви Створити ZIP 
zip -r "%n.zip" "%f"
🗒  Файли Архіви Розпакувати тут 
file-roller --extract-here "%f"
🗒  Файли Зображення Конвертувати у PNG 
convert "%f" "%n.png"
🗒  Файли Зображення Конвертувати у JPG 
convert "%f" "%n.jpg"
🗒  Файли Приховування Зробити файл прихованим 
mv "%f" ".%n"
📂  Папки Термінал Відкрити термінал у цій папці 
xfce4-terminal --working-directory=%d
📂  Папки Термінал Відкрити термінал у батьківській папці 
xfce4-terminal --working-directory="$(dirname "%f")"
📂  Папки Root Відкрити цю папку як root 
pkexec thunar "%d"
📂  Папки Root Запустити root-Thunar 
pkexec thunar
📂  Папки Пошук Catfish у цій папці 
catfish --path="%d"
📂  Папки Інформація Розмір папки 
du -chs "%N" | zenity --text-info --title="Размер"
➿  Шляхи Копіювання Копіювати повний шлях 
echo -n "%f" | xclip -selection clipboard
⚙️  Системні Файлові менеджери Відкрити Midnight Commander 
xfce4-terminal -e "mc %d"
⚙️  Системні Диски Інформація про диски 
lsblk && sudo lsblk -f
⚖️  Разное Інформація Лінк на вищу папку 
ln -s "%f" "%n.link"

⚙️  Набір команд zenity

Розділ Що виконує команда Синтаксис Bash
🛠️ Частина 1. Набір команд zenity 

               
               
           

           
  
               1. Попередження (Кнопка "ОК")
               
               Використовується, щоб показати помилку або важливе повідомлення.
               
               
zenity --warning --title="Увага!" --text="Батарея майже розряджена!"

               
               
           


           
  
               2. Вибір дати (Календар)
               
               Виводить інтерактивний календар. Коли користувач вибирає дату, вона повертається у форматі ДД.ММ.РРРР
               
               
zenity --calendar --title="Виберіть день" --text="Коли дедлайн?"

               
               
           


           
  
               3. Выбор файла на компьютере
               
               Відкриває стандартне системне вікно провідника. Повертає повний шлях до вибраного файлу.
               
               
zenity --file-selection --title="Виберіть шлях до файлу"

               
               
           


           
  
               4. Шкала прогресса (Индикатор загрузки)
               
               Показує повзунок, що заповнюється від 0 до 100%. Дуже круто для довгих скриптів.
               
               
( echo " 10 " ; sleep 1 echo " 50 " ; sleep 1 echo " 100 " ; sleep 1 ) | zenity --progress --title="Копіювання" --text="Будь ласка, зачекайте..." --percentage=0 --width=600 --height=100

               
               
           


            
  
               📦 Частина 2. Мінімальний набір роботи зі Змінними
               
               
               
               

               
               
           


           
  
               У Bash змінні - це дуже просто, але потрібно суворо дотримуватися двох правил:
               
               1. Записати текст або число вручну:
               
               
NAME="Олексій" AGE=30

               
               
           


           
  
               При записі НЕМАЄ знака долара і НЕМАЄ пробілів навколо =
               
               Записати в змінну результат роботи zenity:
               
               
# Обов'язково використовуємо $( ... ) навколо команди MY_FILE=$(zenity --file-selection)

               
               
           


           
  
               Під час читання (виклику) знак долара ОБОВ'ЯЗКОВИЙ.
               
               Використовувати змінну у тексті:
               
               
# Переменная внутри двойных кавычек автоматически превратится в свое значение zenity --info --text="Привет, $NAME! Тебе уже $AGE лет."

               
               
           


           
  
               🧠 Частина 3. Мінімальний набір умов if (Логіка) 
           


           
  
               Умови дозволяють скрипту ухвалювати рішення.(Зверніть увагу: усередині квадратних дужок [...] пробіли після першої дужки і перед останньою обов'язкові!).
               
               Перевірка кнопки Так / Ні (через zenity --question)
               
               
if zenity --question --text="Ви впевнені?"; then
zenity --info --text="Відмінно, продовжуємо!"
else
zenity --info --text="Процес скасовано."
exit 0 # Вихід зі скрипту
fi

               
               
           


           
  
               Очень важная проверка. Флаг -z проверяет, не пустая ли переменная. Если пользователь нажал «Отмена» в окне ввода, переменная будет пустой, и скрипт может сломаться. Защищаем его:
               
               Перевірка: «А чи ввів користувач хоч щось?»
               
               
USER_INPUT=$(zenity --entry --text="Введіть кодове слово:")

if [-z "$USER_INPUT"]; then
zenity --error --text="Ви нічого не ввели або натиснули Скасувати!"
exit 1
fi

               
               
           


           
  
               Якщо потрібно порівняти числа (наприклад, введений час), використовуються спеціальні скорочення:
-gt (Greater Than) - більше >
-lt (Less Than) - менше <
-eq (EQual) - дорівнює =
               
               Порівняння чисел
               
               
CHAIRS=5

if [ "$CHAIRS" -gt 3 ]; then
    zenity --info --text="Стільців більше, ніж три!"
fi

Омніканальність


            


 У сучасному динамічному світі, де технології проникають у всі сфери нашого життя, очікування споживачів кардинально змінилися.
Ми більше не задовольняємося розрізненими точками контакту із сервісом чи документом.

 Саме тут на сцену виходить омніканальність.

 Омніканальність - це стратегія, що поєднує різні канали комунікації в єдину безшовну екосистему, що дозволяє розпочати роботу на одному пристрої, а завершити на іншому без втрати даних та історії.

 Омніканальність – це концепція, яка ставить користувача у центрі всіх взаємодій та процесів. Вона передбачає створення єдиного, безперервного та синхронізованого середовища, де всі канали взаємодії – фізичні та цифрові – працюють в унісон.

 Омніканальна стратегія надає впевненість у тому, що ми отримаємо потрібну інформацію швидко та з необмеженої кількості джерел, а також збережемо ту, що нам потрібна,та ще й з можливістю звернутися до неї з будь-якого місця та з будь-якого пристрою.

 Омніканальна стратегія дозволяє використовувати інформацію як робочий інструмент, одночасно оцінюючи результативність наших дій. Вона створює єдину картину користувача, доступну в усіх точках дотику. 
 Кожен канал збагачує інформацію про документ, ресурс, явище, подію та ін.
Переваги омніканальності незаперечні. 
 Для споживачів це означає зручність, швидкість та передбачуваність, що веде до зростання лояльності та задоволеності.



 Створенням цього проекту я намагався якось організувати традиційний хаос як серед моїх "інформаційних потреб", так і різноманітних сканів, документів, фотографій тощо.
